## 2. Project purpose

Mithril Veil is an open-source, self-hosted Russian PII anonymization/sanitization service for safer LLM workflows.

The goal is to let users locally detect and anonymize sensitive data in Russian texts and documents before sending sanitized content to cloud LLMs such as ChatGPT, Claude, GigaChat, DeepSeek, etc.

Primary target users:

- consultants;
- lawyers;
- appraisers;
- forensic experts;
- accountants;
- small companies;
- private practitioners;
- teams that need a lightweight local alternative to a corporate AI gateway.

Core positioning:

> Mithril Veil: a self-hosted Russian PII anonymization gateway for safer AI workflows.

## 3. Security principles

These principles are mandatory and should not be weakened without explicit architectural review.

### Hard rules

- No real PII in the repository.
- No real documents in the repository.
- No real contracts, court files, valuation reports, bank statements, passports, addresses, client files, or reversible mappings in the repository.
- Only synthetic fixtures are allowed.
- Do not log original input text.
- Do not expose raw detected values in API responses.
- Do not expose raw detected values in CLI reports.
- Do not expose raw detected values in warnings/errors.
- `value_preview` must remain safe, currently always `***`.
- Real user documents must stay local and outside git-tracked paths.
- Cloud agents must not be given access to private documents, `.env`, mappings, or real test corpora.

### Git hygiene

`.gitignore` protects against accidental commits of:

- `.env`
- `.venv`
- `data/private/`
- `data/raw/`
- `data/uploads/`
- `data/runtime/`
- `storage/`
- `logs/`
- SQLite/db files
- mapping files
- document binaries such as PDF/DOCX/RTF
